Introduction

 

Login

You need to login to iMSEP to be authorized to use the maps and data. Login page is like Figure 1.

Figure 1-Login page

In this page:

·        Input your username and password.

·        Check “Remember me” if you want to login automatically the next time visiting iMSEP.

·        Enter any displayed captcha.

·        Click “Log in” button to proceed.

Note:

·        Do not use “Remember me” option in non-secure stations and networks (i.e. Free Wi-Fi, Internet cafés or someone else’s device).

Creating new vector layer

To create a new vector layer click on the button “New Vector Layer”. The property page of new vector layer will be displayed (Figure 12).

Figure 12-New Vector Layer

In the vector data layer property page:

·        Enter a name for new layer

·        Describe this new layer in description text area.

·        In the “Share with others” section you can allow other users to view or edit this layer

·        Define the shape type of vector layer (i.e. Point, Line or Polygon)

·        Define attribute fields

·        Push “Save” button to create new data layer or to apply changes.

To add a new field, click “Add” button (located in fields section) which displays field property dialog (Figure 13).

Figure 13-Field properties

In this dialog:

·        Enter a unique name for field

·        Optionally describe field with more explained texts with Alias ( which is used instead of actual field’ name),Description (which is used to add descriptive information in before the field’s name in edit forms ) and Hint (which is add hint information after field’s input in edit form).

·        Define Group to  classify fields in different panels in the edit form.

·        Define the type of data that will be stored in the field. Supported data types are as follows:

o   Text: variable-length character string, up to defined Length

o   Small Integer: 2 bytes small integer, range: -32768 to +32767

o   Integer: 4 bytes typical choice for integer, range: -2147483648 to +2147483647

o   Big Integer: 8 bytes large integer, range: -9223372036854775808 to +9223372036854775807

o   Real: 4 bytes variable-precision, inexact 6 decimal digits precision

o   Double: 8 bytes  variable-precision, inexact 15 decimal digits precision

o   Numeric: variable number with user-specified precision. Example, A numeric field with Length=5 and Scale=2 can store -999.99 to 999.99

o   Boolean: Boolean values like;true|false,yes|no,1|0

o   Date: 4 bytes date (no time of day)

o   DateTime: 8 bytes both date and time with time zone, example:2018-08-28 12:22:01.673+04:30

·        Set field default value. When there is no entry for field value, this default value will be used otherwise the value will remain undefined.

·        You can choose the field to be a dynamic read-only geometry function such as Area, Perimeter, Length or coordinate of geometry’s center.

·        Use Domain to put constraint on field’s value. You can define value Range or Coded Values to restrict inputs in edit form.

Metadata

Metadata is data that describes other data. Metadata summarizes basic information about data, which can make finding and working with data easier. We use metadata to publish data as OGC services. To create or edit metadata for a specific data layer, in the page “Data layers” click edit button () to navigate to the property page of the data layer. Then in the Metadata section click Create or Edit button to open Edit Metadata page (Figure 18).

Figure 18-Edit Metadata

In this page you can view and edit some of metadata information. The read-only fields are extracted from the data layer automatically. Metadata items are:

Title: data layer main name that reflects data layer’s name.

Alternate title: optional another name of the data layer.

Abstract: is the same as data layer’s description.

Topic Category: Select one or more categories that best describe the data.

Keywords: specify one or more keywords that descript your data.To add new keywords: Type the keyword and press Enter or press “;” character.

Type, Format and Spatial reference: are based on the data layer specifics.

Language: define data’s language.

Creation date: initially is filled based on the date that data layer is created in this system. You can change it to actual date of data creation.

Revision date: is the date of the last changes or update in data.

Creator: define who is responsible for making the content of this data. You can include all creator’s contact information in this field.

Publisher: define who is responsible for making this data available. You can include all publisher’s contact information in this field.

Copyright: describe legal constraints and limitations on the use of this data.

Contributor: define who is responsible for making contributions to the content of this data.

Source: describe source and lineage of data.

Relation: describe any relationship that exists between this dataset and any related resource.

Geographical extent of data: geographical extent of data initially updated from existing data.

Feature styling

Styling is the way to control displaying (or symbolizing) of the features attached to vector layers, such as points, lines and polygons. You can use simple styles for points, lines or polygons or you can use attribute data to symbolize or classify drawn features.

To set style properties of a vector layer use “Style” tab in layer properties. You can employ one of three different display types to style features of the layer:

Simple: All of the features will be displayed with a single style setting. Select “Simple” display type and click on displayed style to modify it.

Unique values: Features will be classified based on the unique values of an attribute field. You may define a separate style for each value. Features that have the same value for that field, will use a common style. For example we can use landuse field in a parcel layer to display each land use type with a specific color. Figure 41 shows an example that style different world cities based on field “status”.

Figure 41- Styling (Unique values)

To create a Unique values styling, select “Unique values” as display type and select the field name, then push “Add all values” to create a style for each value. There is also a “Default style” for values that are not included in value list (for example where the value is null or empty). To modify each style click on it and change its properties.

Range values: Features can be classified based on ranges of numeric values of a field. For example we can classify features represented world cities based on population by defining population ranges. Figure 42 shows an example that classified features of a polygon layer to four groups based on values of field “acres”.

 

Figure 42-Styling (Range values)

To create range values, select “Range values” as display type and select field name. Click on “Create range values” button and enter number of ranges (or classes). Range values will be generated based on minimum and maximum values of the selected field and the number of classes. You can customize generated styles. If for a feature, the value of field is out of defined ranges, then the default style will be used to symbolize it.

Style

A style defined the way that a geometry (point, line or polygon) should be displayed on map. The style properties dialog can be used to modify a style.

 

Figure 43-Style (Simple point)

To set or change colors, type color name or color code or click on colored rectangle to select color visually.

If you need a more dedicated point symbol, select advanced type (Figure 44).

Figure 44-Style (Advanced point)

After selecting symbol, adjust other symbol properties like size, color, border etc. You can put the symbol in a predefined frame and change the default position of symbol by setting Offset X and Y.

 

Figure 45-Style (Line)

Line setting is used to draw lines and the border of polygons. Type or select line color. Use “Width” track bar to define line width. Select type of line cap (Figure 46) which defines the shape of line ends.

Figure 46-Line cap types

Labeling features

Labeling is the process of placing descriptive text onto or next to features on a map. A label is a piece of text on the map that is dynamically placed and whose text string is derived from a feature attribute. To enable or disable labeling for a layer use “Label” tab in layer properties dialog (Figure 56).

 

Figure 56-Labeling features

To enable labeling, check “Label features”. Define the attribute field that text strings will be derived from it. You can use other settings to control displaying of labels.

Use color, size and font to customize labels. There are settings (Outline color and Outline width) that can be used to display a shadow around texts. For line feature you can place texts along line path by selecting “Line” in “Placement” drop down. You can allow labels to exceed the extent of feature shape by setting “Allow overflow”.

In “Source” tab of the layer properties of a WFS layer you can change the url address of the WFS service, feature type and shape type (Figure 60).

Figure 60-Layer source (WFS)

To define the feature type click on “Feature type” input box and clear existing text then type or select one of existing feature types. You can download a GeoJSON or Shapefile version of WFS features. You can copy or convert downloaded features to a new vector data layer. This converted version will be a snapshot copy of remote data. You can query or edit features of this converted data layer.

Note: WFS based features are required to be downloaded first before converting to GeoJSON, Shapefile or local data layer.

Search

We can search features of a vector data layer by defining conditions based on attribute values or spatial locations or both. Clicking  icon located in select toolbar will open search dialog. Figure 65 shows a sample for searching for a park (or parks) where the value of attribute “name” is ‘Brooklyn Park’.

 

Figure 65-Search

To search by attribute you must create a logical expression based on attribute names and value. To create this expression you can use “Field”, “Operator” and “Value” lists. After defining these inputs, push “Add to expression” button to add a rule to “Expression”. You can combine multiple rules to extend the expression. By default rules are combined by “AND” logical operator, but you can manually edit the expression.

Figure 66-Search (Logical expression)

Figure 66 shows an example where we want to find all features of layer “Park.shp” with area beyond 20 to 50 acres range. In this case we can use the following expression:

NOT ( (acres > 20) AND (acres < 50) )

Also, we can use:

(NOT (acres > 20) ) OR (NOT (acres < 50) )

Or:

(acres <= 20) OR (acres >= 50))

Any combination of attribute names, values and logical or mathematical operators can be used to build the expression (Figure 67). 

Figure 67-Search (Relationship between attribute values)

For text values we must enclose text string in two single quotation marks (‘). The operator “LIKE” is used for partial comparison of texts. This operator is used to match text values against a pattern using wildcards. If the search expression can be matched to the pattern expression, the LIKE operator will return true.

There are two wildcards used in conjunction with the LIKE operator :

·        The percent sign (%)

·        The underscore (_)

The percent sign represents zero, one, or multiple characters. The underscore represents a single character. For example:

(name LIKE 'Al%')

 Will select all features that have name starts with “Al”. These symbols can be used in combinations. If either of these two signs is not used in conjunction with the LIKE clause, then the LIKE acts like the equals operator. The text comparison process is case sensitive so if you want to select all combination of upper case and lower case characters you must use an expression like following:

(LOWER(name) LIKE 'al%')

In this case we use LOWER function to change the value of “name” to lower case and then compare it to ‘al%’ string. You can use other PostgreSQL String functions in search expression.

Note: if you want to use LIKE operator for numerical values you need to explicitly cast values to text strings using “::text” expression. For example:

(shape_area::text LIKE '10___')

Will match shape_area with values from 10000 to 10999.

Raster analysis

Analysis tasks that you can perform on raster data layers are collected in “Analysis toolbar” . You must have required privileges to execute these tasks.

Slope

Slope task identifies the slope (rate of maximum change in elevation) from each cell of a DEM raster. This tool only works on rasters of type DEM so if you have a single-band raster that is not defined as DEM you must first change its type to single-band DEM in data layer properties page.

For each cell, the Slope tool calculates the maximum rate of change in value from that cell to its neighbors. The tool fits a plane to the values of a 3 x 3 cell neighborhood around the processing or center cell. The slope value of this plane is calculated using the average maximum technique. The lower the slope value, the flatter the terrain; the higher the slope value, the steeper the terrain.

The rates of change of the surface in the horizontal (dz/dx) and vertical (dz/dy) directions from the center cell determine the slope. The basic algorithm used to calculate the slope is:

 slope_radians = ATAN ( √ ([dz/dx]2 + [dz/dy]2) )
 

The values of the center cell and its eight neighbors determine the horizontal and vertical deltas. The neighbors are identified as letters from a to i, with e representing the cell for which the slope is being calculated.

The rate of change in the x direction for cell e is calculated with the following algorithm:

  [dz/dx] = ((c + 2f + i) - (a + 2d + g) / (8 * x_cellsize)
 

The rate of change in the y direction for cell e is calculated with the following algorithm:

  [dz/dy] = ((g + 2h + i) - (a + 2b + c)) / (8 * y_cellsize)

 

Note: For more information about Slope, please refer to ESRI - How Slope works

The output slope raster can be calculated in three types of units; degrees, radians or percent.

To execute, select “Slope” from analysis toolbar (Figure 89).

Figure 89-Slope

 

In the Slope dialog, select units type and click OK.

Figure 90-Slope (sample output)

Hillshape

Creates a shaded relief from a surface raster by considering the illumination source angle and shadows. It does this by setting a position for a supposed light source and calculating the illumination values of each cell in relation to neighboring cells.

Note: For more information about Hillshade algorithm, please refer to ESRI - How Hillshade works

This tool only works on rasters of type DEM so if you have a single-band raster that is not defined as DEM you must first change its type to single-band DEM in data layer properties page.

To execute, select “Hillshade” from analysis toolbar (Figure 91).

Figure 91-Hillshade

You can define Azimuth angle and Altitude angle of light source.

The azimuth is expressed in positive degrees from 0 to 360, measured clockwise from north.

The altitude is expressed in positive degrees, with 0 degrees at the horizon and 90 degrees directly overhead.

Click OK to start hillshade generating process.

Figure 92-Hillshade (sample output)

Classify

Reclassifies or changes the values in a raster into a new raster.

To execute this task click “Classify” in the analysis toolbar.

Figure 95-Classify

In the classify dialog:

Select the band of raster that will be used for classification.

Define conversion map by specifying ranges.

Each range definition consists of four number, two values from origin raster and two values in output raster.

For single values leave “To” input field empty.

Select “Integer” or “Float” for output data type.

For example in Figure 95 all values from 1120 to 2000 (greater than or equal 1120 and less than 2000) will be converted to 1, values from 2000 to 3000 (greater than or equal 2000 and less than 3000) will be converted to 2 to 3 respectively. All values greater than or equal 3000 and less than 4000 will be converted to single value 4. Figure 96 shows two different outputs with Float and Integer data types for this example.

Catalog

CSW (Catalog Service for Web) is one of the OGC service that is implemented in this system. This service provide a standard interface for searching among resources in this system.

Catalog page is designed to simplify searching through this CSW or any other registered third-party CSWs, for users (Figure 100).

Figure 100-Catalog

To search for available published resources you can type searching keyword in the search box and press Enter or hit  button. The result will be displayed in right panel.

You can search other registered CSWs by selecting them in “Search in” drop-down below search box. Specify other filter conditions based on Type, Topic, Keywords, Contributor, Creation date, Revision date and Geographical extent of data to find more specific results.

Clicking on the name of each result item will display more metadata info about that item. Result items may have one or more OGC services that you can use in any supporting software or system. You can also check this services in the current system by clicking “View service in a map”.
